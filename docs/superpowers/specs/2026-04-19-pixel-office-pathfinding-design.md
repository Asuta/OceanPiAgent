# Pixel Office Pathfinding Design

## Summary

为 `/world` 像素办公室加入轻量级网格寻路。Agent 从休息区走向工位、或在休息区内换目标点时，不再直接用单段直线移动，而是基于办公室静态障碍物进行 A* 绕行，并用连续匀速插值沿路径前进。

## Goals

- 小人移动看起来更像真实 2D 游戏，而不是点到点瞬移
- 桌子、椅子、电脑等工位家具视为不可穿透障碍
- Agent 从障碍一侧移动到另一侧时具备稳定绕行能力
- 保持当前 React/CSS 架构，不引入 Phaser 或重型导航系统

## Non-Goals

- 不做 Agent 与 Agent 的动态避让
- 不做连续空间 navmesh
- 不做地图编辑器或可配置家具摆放
- 不改变后端 runtime working/resting 语义

## Recommended Architecture

### 1. 静态世界布局层

新增前端纯数据模块，集中定义：

- 世界边界
- 网格分辨率
- 工位站位点
- 休息区漫游点
- 障碍物矩形（桌面、椅子、电脑占位）

这层只描述“地图是什么”，不承担状态推导和运动插值。

### 2. 寻路层

新增纯函数寻路模块，负责：

- 世界坐标和网格坐标互转
- 根据障碍物生成 walkable grid
- 使用 A* 求解路径
- 路径压缩，去掉多余共线拐点
- 输出世界坐标路径点

### 3. 运动层

新增客户端 hook 管理每个 Agent 的短时运动状态：

- 当前实时位置
- 当前路径与路径段索引
- 当前段开始时间
- 速度
- 当前逻辑目标 key

当 working/resting 状态变化，或休息区目标点刷新时，运动层从“当前位置”重新求路，而不是从上一个逻辑锚点硬跳。

### 4. 渲染层

`AgentWorldPanel` 只消费：

- 逻辑 snapshot（状态、气泡、房间信息）
- 运动层返回的实时位置

桌子、椅子、电脑继续由 DOM/CSS 渲染，但它们的布局数据与障碍物数据来自同一份静态世界定义，避免视觉和碰撞不一致。

## World Rules

- resting 状态：在休息区漫游点之间移动
- working 状态：移动到自己的工位站位点，而不是桌子中心
- 新目标生成后，先做路径求解，再开始移动
- 如果目标不可达，回退到最近的可达邻近点
- 20 秒工具 grace 仍保留；grace 期间也按 working 路径逻辑处理

## Pathfinding Choice

采用低分辨率 2D 网格 + A*：

- 足够轻量
- 能可靠绕过静态障碍
- 实现成本适合当前 React 页面

为避免“棋盘感”过强，路径求出后需要：

- 压缩连续共线节点
- 用恒定速度沿段移动

## Testing

- A* 能绕过单个工位障碍
- 路径不会穿过桌子/椅子障碍格
- 不可达目标会回退到可达点
- working / resting 切换会生成新路径
- 连续帧推进时位置单调前进，不会跳点
- world 页集成后仍能显示气泡、状态和工位停留逻辑

## Risks

- 如果网格过密，计算量会变大；过疏则路径难看
- 如果视觉桌子位置和障碍数据漂移，会出现“看起来能走但走不过去”
- 如果每次状态变化都重算整张地图，未来家具增多时会浪费计算，应优先缓存静态 grid

## Decision

按“静态布局 + A* + 运动 hook + 现有 DOM 渲染”的方案开发一期。
