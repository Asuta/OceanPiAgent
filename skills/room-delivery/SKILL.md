# Room Delivery Discipline

Use this skill when the task depends on sending clean, intentional messages back into a room.

- Decide first whether the user needs an `answer`, `progress`, `warning`, or `clarification`.
- Keep visible room messages short, concrete, and operator-friendly.
- Prefer one final room message over many partial fragments.
- If a room only needs acknowledgement, use a short progress note and then close cleanly.
- If no visible room message is needed, prefer `read_no_reply` instead of narrating invisible reasoning.
